# Expansion Roadmap

## Expansion Principle

Only add a mechanic when the previous layer is readable, testable, and represented in the legal-action protocol.

Each expansion should include:

- rule implementation
- UI representation
- selected-panel explanation
- event log messages
- protocol payload changes
- tests
- balance notes

## Recommended Order

1. Reinforcements
2. Basic income
3. Stability
4. Decrees
5. Influence support/pressure
6. Corruption and siphoning
7. Region cards
8. Set cash-ins
9. Counter-draft
10. Victory warnings
11. Mandates
12. Stronger bots
13. Browser Agent Console polish
14. Localhost bridge
15. Optional backend/shared matches

## Why This Order

Reinforcements extend the board without adding hidden information.

Income and stability add Domain stakes.

Decrees add engine-building after Domains are meaningful.

Influence and corruption add pressure once stability exists.

Cards and counter-draft add hidden planning after public systems are readable.

Mandates should come late because they depend on the rest of the game being legible.

## Expansion Gate

Before adding a new layer, answer:

- Can players understand the current board state?
- Can the engine explain legal actions?
- Can the selected panel explain the mechanic?
- Can a bot use the mechanic through legal actions?
- Can tests cover the mechanic without UI?
- Does the portfolio story improve?

If the answer is no, do not add the layer yet.
